# 日志生成 Skill

## 目的

这份 skill 负责把当天已经收集到的真实信息，整理成一篇可读的日记。

它不凭空编造内容，只基于已有输入做结构化生成：

- 今日日志已有内容
- 随手记里的当天条目
- 已写入的数据记录
- 已知的睡觉 / 起床 / 训练 / 饮食 / 完成事项

## 适用场景

- 当天日志结构还是空的
- `今日变化` 只有碎片句子
- `数据记录` 只有零散信息
- `复盘` 还没写
- 归档完随手记后，需要把日志整理成正式版本

## 输出目标

生成并维护三块：

### 1. `今日变化`

- 只写当天真实发生的事
- 用 3 到 6 条 bullet
- 优先写主线推进、训练、饮食、作息、恢复、重要事件
- 不把明天计划误写成今天已完成

### 2. `数据记录`

- 保持 Markdown 表格
- 每行只写一个明确指标
- 格式固定：
  - `维度`
  - `指标`
  - `数值 / 状态`
- 没有把握的数值不乱填

### 3. `复盘`

固定写三句：

- `做得好的地方：...`
- `需要修正的地方：...`
- `下一步：...`

要求：

- 基于当天事实
- 短、准、能执行
- 不空泛喊口号

## 生成原则

- 先保真，再美化
- 不把未来计划包装成已发生事实
- 不为了凑字数写流水账
- 不凭空补不存在的训练量、热量、学习时长
- 如果已有内容比模型生成更具体，保留已有事实

## 输入源优先级

1. `logs/daily/YYYY-MM-DD.md`
2. `inbox/quick-capture.md` 中当天条目
3. 已有结构化记录
4. 其他当天明确写入系统的内容

## 生成后的理想状态

一篇好的日记应该做到：

- 顶部 `今日变化` 能让你 15 秒回想起这一天
- `数据记录` 能看出变化
- `复盘` 能直接指导明天

## 推荐接入点

- 随手记归档完成后自动触发
- 或者手动对某一天单独执行一次整理

## 不做的事

- 不生成长篇抒情散文
- 不覆盖明确手写的重要事实
- 不把同一条信息重复塞到三个 section
