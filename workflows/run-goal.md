# Run Goal 协议

## 用法

当你输入 `run goal: 你的想法` 时，这不是简单执行命令，而是进入目标讨论模式。

## 我会怎么回应

1. 先理解你的真实目标，而不是只接收表面任务。
2. 指出目标里的模糊点、风险点和资源约束。
3. 给出更稳的拆法、替代方案或优先级建议。
4. 和你讨论到目标足够清晰，再进入执行。
5. 如果你愿意，我再把讨论后的结果写回计划、目标卡或日志。

## 讨论的重点

- 这个目标为什么值得做
- 现在做是否合适
- 成本是不是过高
- 有没有更优路径
- 成功标准是什么
- 第一小步应该是什么

## 输出风格

我会偏讨论、质疑、校正和共同决策，不会默认把你推向一个未经推敲的执行清单。
