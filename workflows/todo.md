# Todo 协议

## 用法

当你输入 `todo: 要做的事情` 时，系统默认把它视为一个待推进事项，而不是普通聊天内容。

## 我会怎么处理

1. 提取这件事的核心动作。
2. 判断它更适合进入日、周还是月层级。
3. 识别它属于哪个维度。
4. 给它补标签：一级标签默认用八维，二级标签用具体主题。
4. 需要时补一条建议：先做什么，或者是否值得做。
5. 为这条 todo 生成独立 `.md` 文件，给出文件 id 和路径。
6. 再把它写进合适的位置，例如当前聚焦、列表或计划。

## 适用场景

- 记录一个要做的事情
- 把临时想法转成待办
- 给计划系统补新事项

## 默认输出风格

偏收纳、归类和轻量建议，不会展开成长篇讨论。

## 当前文件规则

- todo 文件放在 `todos/`
- 每条 todo 都有唯一 id
- 默认命名格式：`todo-YYYY-MM-DD-slug`
- 你后续补充内容时，我应该优先写回原 todo 文件
- 如果你刚创建完一个 todo，又继续补充同主题内容，默认不要新建，而是回写刚才那个 todo
- 对于“购买”类输入，默认不要继续新建散的 todo，而是优先更新 `购买总表`
- 标签规则默认使用两层：
- 一级标签：八维
- 二级标签：更具体的子领域，例如 `篮球 / 羽毛球 / 摄影 / 饮食 / 和猫`
