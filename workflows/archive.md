# Archive 协议

## 用法

当你输入 `archive: ...`，或者点击首页 `随手记` 的【归档】按钮时，系统默认把它视为一次归档动作。  
它不是简单搬运，而是把当天随手记里的内容整理后，分发到正确的系统文件里。

## 目标

让随手记不只是临时堆积。  
归档之后，内容应该进入真正长期维护的位置，例如：

- Todo
- 日程
- 领域
- 今日日志
- 周志
- 和猫相关索引
- 购买总表 / 衣柜 / 资产

## 归档对象

默认归档范围是“今天的随手记”。

来源文件：

- `inbox/quick-capture.md`

默认只处理：

- 今天新增的文字
- 今天新增的图片
- 今天新增的 URL

## 我会怎么处理

1. 先读取今天的随手记条目。
2. 判断每条内容属于：
   - 只是日志材料
   - 可以推进成 Todo
   - 可以推进成日程项
   - 可以更新现有 Todo
   - 可以更新某个领域
   - 可以写进购物 / 衣柜 / 纪念日 / 和猫索引
3. 尽量优先写回已有文件，不重复造新文件。
4. 把当天关键变化提炼进：
   - `logs/daily/YYYY-MM-DD.md`
   - 必要时 `logs/weekly/YYYY-WXX.md`
5. 归档完成后，应当给出一份简短说明：
   - 写到了哪些文件
   - 新建了哪些内容
   - 哪些内容还只是留在随手记里，没有足够信息归档

## 归档联动规则

### 1. Todo 联动

如果随手记内容明显是“要做的事”：

- 优先匹配已有 Todo
- 如果命中已有 Todo：
  - 回写对应 `todos/*.md`
- 如果没有命中，但确实是一个明确待办：
  - 新建 Todo
  - 同步 `todos/index.md`

### 2. 领域联动

如果随手记内容可能属于某个长期子领域，例如：

- 篮球
- 足球
- 羽毛球
- 护肤
- 穿搭
- 摄影

默认先调用本地 `codex exec` 做结构化判断，再更新对应 `iterations/*.md` 的：

- `里程碑`
- `成就归档`
- `最近变化`
- `当前问题`
- `经验沉淀`
- `下一步`

判断原则：

- 如果是“已经形成阶段结果”的内容，优先写 `里程碑` 或 `成就归档`
- 如果只是当天观察、经验或状态变化，优先写 `最近变化` / `经验沉淀`
- 不把长期工作的阶段成果只留在日记里

当前判断不是前端猜，也不是关键词硬匹配，而是走本地 Codex skill：

- `scripts/codex-route-capture.js`

## 2.5 日程联动

如果随手记内容明显是：

- 周几 + 时间段 + 事项
- 日期 + 时间段 + 事项
- 今天 / 明天 + 时间段 + 事项

默认同步到：

- `schedules/weekly-schedule.md`

例如：

- `周四晚上 去东操音乐跑`
- `明天下午 做 3 小时科研`
- `04-12 晚上 和猫去看电影`

## 3. 日志联动

只要随手记内容属于今天真实发生的事，默认写进：

- `logs/daily/YYYY-MM-DD.md`

写入位置优先是：

- `今日变化`

如果内容足够结构化，再补到：

- `数据记录`
- `复盘`

### 4. 周志联动

如果今天的随手记里有对周推进有意义的事情，例如：

- 完成了一项任务
- 出现了明显变化
- 某个方向有阶段性推进

才提炼进：

- `logs/weekly/YYYY-WXX.md`

不是所有随手记都进周志。

### 5. 和猫联动

如果内容里出现和 `猫` 有关，并且属于：

- 吃饭
- 玩乐
- 约会
- 特殊活动

默认同步到：

- `private/cat-food-fun-index.md`

如果明显是高光时刻，再同步到：

- `private/cat-100-things.md`

### 6. 购买 / 衣柜 / 资产联动

如果内容和购买、新增物品、护肤品、衣服、鞋有关：

- 购买计划优先同步到 `todos/2026-04-06-purchase-master.md`
- 已拥有的东西优先同步到 `lists/wardrobe.md` 或 `assets/asset-register.md`

## 不应该做的事

- 不要把每一条随手记都机械地变成 Todo
- 不要重复新建多个同主题文件
- 不要为了“归档”而制造大量冗余摘要
- 不要覆盖掉原始随手记，除非明确标记已归档状态

## 当前推荐策略

归档默认分两层：

1. 原始材料仍保留在 `inbox/quick-capture.md`
2. 提炼后的结果再写进对应正式文档

这样可以保证：

- 原始信息不丢
- 正式系统保持整洁

## 按钮规则

首页的【归档】按钮未来默认执行：

- 读取今天的随手记
- 自动分类
- 自动联动 Todo / 领域 / 日志 / 私人索引
- 输出一份归档结果说明

如果某条内容信息不足：

- 先保留在随手记
- 不强行归档
- 在结果里提示“需要补充”
