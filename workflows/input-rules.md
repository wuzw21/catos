# 输入 API 规则

## 目的

这份文档定义当前系统支持的输入协议。  
目标是让你以后输入一句话时，系统知道应该把它当成目标讨论、待办、完成记录、网络探索，还是当天日志字段更新。

## 先区分两种入口

### 1. 直接和我对话

用途：

- 讨论
- 分析
- 比较方案
- 追问
- 让系统直接正式写入

默认特点：

- 更偏“理解你的意图”
- 更适合 `run goal`、计划讨论、科研推进、策略判断
- 不一定先进收集箱
- 如果你明确说 `帮我记录` / `todo` / `eat` / `finish`，我会直接走正式写入链路

### 2. 首页 `随手记`

用途：

- 先丢原始材料
- 收集临时想法、事实、图片、零碎信息
- 稍后再归档

默认特点：

- 先进入 `inbox/quick-capture.md`
- 再通过归档 / 沉淀动作分发到日志、Todo、日程、行动领域、和猫索引等正式位置
- 更适合“先记住，后整理”

一句话区分：

- 直接对话 = 跟我一起想
- 随手记 = 先把东西丢进系统

## 标签默认规则

- 一级标签：固定用八维
- 二级标签：用具体子领域
- 常见二级标签例如：
- `篮球`
- `足球`
- `羽毛球`
- `摄影`
- `饮食`
- `护肤`
- `和猫`

## 一级协议

### `run goal: ...`

用途：

- 讨论一个目标
- 不直接执行
- 先分析值不值得做、怎么做更稳

默认处理：

- 进入讨论模式
- 输出风险、约束、更优路径和第一小步

目标文件：

- `workflows/run-goal.md`

### `todo: ...`

用途：

- 记录一个要做的事情
- 新增待办
- 补计划事项

默认处理：

- 判断是日 / 周 / 月层级
- 判断属于哪个维度
- 生成独立 todo 文件
- 写入当前聚焦、列表或计划

目标文件：

- `workflows/todo.md`

### `finish: ...`

用途：

- 记录一件完成的事情
- 记录实际推进

默认处理：

- 如果指向已有 Todo，则自动改对应 Todo 文件状态
- 同步 `todos/index.md` 和前端数据
- 写入今日日志
- 需要时提炼到周志
- 需要时同步到相关维度 `changes.md`

目标文件：

- `workflows/finish.md`

### `research: ...`

用途：

- 发起网络探索
- 调研一个主题、攻略或资料

默认处理：

- 先理解探索目标
- 需要时去网络搜索
- 总结为简明结论
- 值得长期维护时新建探索文档

目标文件：

- `workflows/research.md`

### `archive: ...`

用途：

- 归档随手记
- 把当天临时记录整理进正式系统
- 自动联动 Todo、领域、日志和私人索引

默认处理：

- 读取今天的 `随手记`
- 识别哪些内容应当进入 Todo、领域、日志、购买总表、和猫索引等
- 优先写回已有文件
- 需要时新建 Todo
- 输出归档结果和写回位置

目标文件：

- `workflows/archive.md`

### `eat: ...`

用途：

- 记录吃了什么
- 进入饮食与热量记录流程

默认处理：

- 写入饮食记录
- 尽量估热量和蛋白质
- 判断是正餐、加餐还是宵夜

目标文件：

- `workflows/eat.md`

### `train: ...`

用途：

- 记录训练、运动或恢复活动

默认处理：

- 写入今日日志
- 同步健身 / 运动相关变化
- 保留训练结构

目标文件：

- `workflows/train.md`

### `spend: ...`

用途：

- 记录花钱和消费

默认处理：

- 写入理财记录
- 判断类别和金额

目标文件：

- `workflows/spend.md`

### `mood: ...`

用途：

- 记录情绪和精力状态

默认处理：

- 写入生活状态
- 记录对执行和恢复的影响

目标文件：

- `workflows/mood.md`

## 二级快捷记录

这些输入不是独立协议文件，但已经纳入系统规则，默认按当天日志字段处理。

### `帮我记录：...`

用途：

- 用自然语言补一段当天真实发生的事

默认处理：

1. 写入当日日志
2. 提炼周级关键进展时同步到周志
3. 同步到相关维度变化

适合：

- 一整段真实生活描述
- 包含运动、吃饭、科研、学习、社交等混合事件

### `sleep: HH:MM` / `睡觉：HH:MM`

用途：

- 记录当天入睡时间

默认处理：

- 更新当日日志中的 `生活技能 | 睡觉时间`
- 如果已有起床时间，则自动计算总睡眠时长

### `wake: HH:MM` / `起床：HH:MM`

用途：

- 记录当天起床时间

默认处理：

- 更新当日日志中的 `生活技能 | 起床时间`
- 如果已有睡觉时间，则自动计算总睡眠时长

## 饮食记录规则

当前优先使用：

- `eat: ...`
- 或 `帮我记录：...` 中包含吃饭信息

默认处理：

- 写入当日日志里的 `生活技能 | 饮食 / 恢复`
- 如果信息足够，会进一步估热量和蛋白质

## 长期领域补充规则

如果你想从随手记里显式补一个长期工作的阶段节点，当前推荐直接写：

- `【里程碑】篮球：连续 7 天完成晚间投篮练习`
- `【里程碑】足球：第一次踢进稳定轮换`
- `【里程碑】健身：卧推 70kg x 5`

默认处理：

1. 优先识别对应 `iterations/*.md`
2. 写进该领域的 `里程碑`
3. 同时保留到当日日志 / 随手记原文

`成就` 默认不需要你手写。

当前规则：

- 归档时会尝试从值得回看的领域变化里，自动生成一条简洁的 `成就归档`
- 如果你想强制指定，也可以写：
  - `【成就】篮球：第一次明显感觉出手开始稳定`

## 默认路由规则

### 如果有明确前缀

- 优先按前缀协议处理

### 如果有明确文件或 id

- 优先写回那个已有文件

### 如果没有前缀，但像真实记录

- 默认按 `帮我记录：...` 理解

### 如果没有前缀，但像目标讨论

- 默认建议改写成 `run goal: ...`

### 如果没有前缀，但像待办

- 默认建议改写成 `todo: ...`

## 当前随手记路由表

| 你输入的内容 | 系统怎么判断 | 默认出口 |
| --- | --- | --- |
| 临时想法 / 图片 | 先收集，不立刻正式归档 | `inbox/quick-capture.md` |
| 今天真实发生的事 | 当天事实 | `logs/daily/YYYY-MM-DD.md` |
| 对本周有推进价值的事 | 周级推进 | `logs/weekly/YYYY-WXX.md` |
| 今天 / 今晚 + 事项 | 强规则，直接视为今日日程 | `schedules/weekly-schedule.md` 里的今天 |
| 时间 + 动作 | 日期 / 周几 / 时间段 + 事项 | `schedules/weekly-schedule.md` |
| 明确待办 | Todo 语义 | `todos/*.md` |
| 购买 / 待买 | 买、购买、下单、待买 | `todos/2026-04-06-purchase-master.md` |
| 篮球 / 足球 / 健身 / 羽毛球 / 护肤经验 | 命中行动领域关键词 | `iterations/*.md` |
| 和猫吃饭 / 玩 / 去哪里 | 普通活动索引 | `private/cat-food-fun-index.md` |
| 和猫值得回看的特殊活动 | 高光活动 | `private/cat-100-things.md` |

说明：

- 如果句子里有 `今天 / 今晚 / 今早 / 今日`，系统优先把它当成“今日日程”处理。
- 行动领域沉淀现在支持关键词直达，命中后优先直接写入对应领域。
- 当前不把 `asset` 作为高频输入协议。
- `科研 / 学习` 的归档词库主存见：
  - `dimensions/03-research/rules.md`
  - `dimensions/04-learning/rules.md`
  - 汇总说明见 `notes/research-keyword-lexicon.md`

### 如果没有前缀，但像对刚创建 todo 的补充

- 默认写回最近同主题 todo 文件，不新建

## 特殊语义规则

- `猫` 默认指代女朋友。
- `asset` 指你已经拥有的资源，不只指金钱。
- 如果输入里明确出现和 `猫` 一起吃饭、出去玩、约会或体验活动，默认把日期和描述追加到 `private/cat-food-fun-index.md`。
- 如果这件事明显属于高光时刻或特殊活动，再额外同步到 `private/cat-100-things.md` 的 `Shining Time`。

## 路由文档

- 更详细的“自动写到哪个 md”规则见 `workflows/routing-map.md`
- 首页、随手记、购买、资产、日程和领域沉淀的统一分流 skill 见 `workflows/home-and-capture-routing-skill.md`

## 当前支持总表

| 输入 | 类型 | 默认动作 |
| --- | --- | --- |
| `run goal: ...` | 目标讨论 | 讨论、分析、再决定执行 |
| `todo: ...` | 待办记录 | 生成独立 todo 文件，再写入计划 / 列表 / 当前聚焦 |
| `finish: ...` | 完成记录 | 若命中 Todo 则先改 Todo 状态并同步索引 / 前端数据，再写入日志 / 周志 / 维度变化 |
| `research: ...` | 网络探索 | 搜索、总结、必要时建探索文档 |
| `eat: ...` | 饮食记录 | 记录饮食并估热量 / 蛋白质 |
| `train: ...` | 训练记录 | 写入训练 / 运动 / 恢复 |
| `spend: ...` | 财务记录 | 写入金额 / 类别 / 备注 |
| `mood: ...` | 状态记录 | 写入情绪 / 精力 / 影响 |
| `asset: ...` | 资产记录 | 写入资产总表与八维归属 |
| `帮我记录：...` | 自然语言记录 | 自动拆到日志 / 周志 / 维度 |
| `sleep: ...` / `睡觉：...` | 日志字段 | 更新睡觉时间 |
| `wake: ...` / `起床：...` | 日志字段 | 更新起床时间并计算睡眠 |
| `周四晚上 去东操音乐跑` 这类自然句子 | 日程插入 | 自动拆成日期 / 分段 / 事项，写入 `schedules/weekly-schedule.md` |

## 后续扩展位

后面可以继续加：

- `asset: ...`
- `journal: ...`
- `relationship: ...`
- `photo: ...`

但只有在你真的高频使用时再加，不提前堆协议。
