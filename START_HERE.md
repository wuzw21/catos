# Start Here

这份文档用于第一次启动这个项目。

## 1. 初始化内容目录

如果你只是想先跑仓库自带的公开示例：

```bash
npm run init:demo
```

然后启动：

```bash
npm run serve:demo
```

打开：

```text
http://127.0.0.1:2333/web/index.html
```

## 2. 如果你要用自己的私有数据

先初始化一个你自己的内容目录：

```bash
node scripts/init-content-root.js /absolute/path/to/your-content
```

再启动：

```bash
PEOS_CONTENT_ROOT=/absolute/path/to/your-content npm run serve
```

## 3. 第一次最先补什么

优先补这三类：

1. `soul.md`
2. `profile/personal-info.md`
3. 首页今天的内容

## 4. 如何初始化 `soul.md`

最推荐的方式不是手改一堆表，而是直接和 Codex 对话。

你可以直接说：

```text
帮我初始化 soul：
男，23岁，博士生，AI研究方向。
当前体重 80kg，体脂率 20%，存款 5w。
接下来一年主线是科研、运动、生活系统建设。
```

系统应该把这些信息整理写进：

- `soul.md`
- 必要时同步 `profile/personal-info.md`

你也可以先参考 `content.example/` 里的空模板目录，再逐步写成你自己的内容。

## 5. 后续怎么记录

### 适合直接对话的

- `todo: 购买新篮球鞋`
- `finish: 完成了 3 小时研究`
- `research: 研究篮球出手训练方法`
- `sleep: 01:30`
- `wake: 11:30`
- `eat: 中午吃了健身餐`

### 适合首页随手记的

- 临时想法
- 一句事实
- 一张照片
- 一个待会再整理的碎片

然后点首页里的：

- `保存随手记`
- `归档今天随手记`

## 6. 一个最顺的启动顺序

1. 初始化内容目录
2. 打开首页
3. 用对话先补 `soul.md`
4. 用首页写今天的日计划
5. 用随手记开始丢原始材料
6. 再让系统帮你归档到日志 / Todo / 领域

## 7. 当前限制

当前系统最适合：

- 日常持续维护
- 随手记归档
- 结构化回写

第一次的大规模身份初始化，仍然更适合通过直接对话完成，而不是仅靠首页随手记。

## 8. 公开仓库里的模板是什么

`content.example/` 现在是一套空模板内容目录。

它保留了：

- 文件结构
- 路由方式
- 首页到日志的联动关系
- 第一次打开时的引导提示

它不包含：

- 作者的真实日志
- 作者的私人关系细节
- 真实消费、照片和长期轨迹
