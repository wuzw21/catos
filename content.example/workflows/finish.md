# Finish 协议

## 用法

当你输入 `finish: 完成的事情` 时，系统默认把它视为一个已完成事件。

## 我会怎么处理

1. 提取完成了什么。
2. 判断它应该进入今日日志、周志还是某个维度变化记录。
3. 如果这件事对目标推进有意义，会补一句“它意味着什么变化”。
4. 如果它对应一个已存在的 Todo，我应该自动把对应 Todo 文件状态改成 `已完成`，并同步 `todos/index.md` 与前端数据。
5. 把记录写回系统，而不是只回复一句“收到”。

## 适用场景

- 完成一项作业
- 完成一次训练
- 完成一个阶段任务
- 完成某个生活维护动作

## 默认输出风格

偏记录、归档和提炼，不额外拔高意义。

## Todo 联动规则

- 如果 `finish:` 指向一个已有 Todo 标题或 Todo ID，默认优先回写那个 Todo。
- 默认把 Todo 状态改成 `已完成`。
- 同时同步 `todos/index.md` 和 `web/data.js`。
- 然后再同步今日日志、周志和相关维度变化。
- 如果内容是“买到了 / 下单了 / 购买完成了某个东西”，默认优先更新 `todos/2026-04-06-purchase-master.md` 对应条目。
- `购买总表` 是统一入口；单个购买项完成，不代表整个购买总表必须立刻结束，除非你明确说“finish: 购买总表”。
