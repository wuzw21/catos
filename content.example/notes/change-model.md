# 变化可视化模型

## 核心原则

我们的核心不是写很多计划，而是让“变化”能被看见、被比较、被验证。

## 一套最小可用结构

### 1. 基线

先记录起点，不然之后没有变化可比。

- 科研：当前论文主线、每周深度工作频次、当前实验阶段。
- 运动：体重、体脂率、卧推、深蹲、硬拉、引体向上。
- 金融：当前存款 5w。
- 生活技能：作息、护肤状态、穿搭投入、做饭能力、桌搭状态。
- 音乐 / 运动 / 摄影：当前水平和是否已有稳定训练。

### 2. 目标

每个目标都要写成 `起点 -> 目标值 -> 时间范围`。

例如：

- 体脂率：20% -> 15% -> 2027-03-11
- 存款：5w -> 10w -> 第一阶段
- 引体向上：3 个 -> 20 个 -> 年度目标

### 3. 过程日志

每天和每周记录真实动作，不记录空话。

- 今天练了什么
- 今天研究推进了什么
- 今天支出了什么
- 今天是否做了护肤、记账、拍照、穿搭研究

### 4. 周期快照

每月做一次快照，把关键指标重新写一遍。

- 当前值
- 相比上个月的变化
- 是否接近目标
- 哪些变化是真实发生的

### 5. 证据材料

变化最好有证据，不只是一句“我变好了”。

- 照片：体态、穿搭、桌搭、摄影作品
- 数据：体重、力量、存款
- 产出：论文、笔记、实验记录
- 作品：音乐练习、摄影作品

## 以后在网页里要直接显示什么

为了让变化一眼能看见，核心卡片最好显示：

- 起点
- 当前值
- 目标值
- 最近一次变化
- 变化幅度
- 更新时间

## 判断这个系统是否有效

如果一个月后你能回答下面这些问题，说明系统有效：

1. 我比 2026-04 起点具体进步了什么？
2. 哪个目标是真的在推进，哪个只是嘴上在说？
3. 哪个习惯连续发生了，哪个只是偶尔出现？
4. 有没有照片、数据或产出证明我的变化？
